Objects

  • Axioms and Rules are basic types of objects in our system. Both are created by the keyword: new.

    Axioms

    • new Axiom(params string[] goals)
      goals
      goals

      Description

      Example

      new Axiom("Tall", "Cylinder");
      • This means that a default shape is created. Its left-lower-back corner is placed at (0,0,0).
      • It has the form of a unit box (sized 1).
      • "Tall" and "Cylinder" are Goals. It means that we want the shape to become a tall cylinder and we will be searching or writing rules to fulfill these goals.
      • Note that goals and tags are case insensitive. It makes no difference whether you write "Cylinder", "CYLINDER", "cYLINDEr" or "cylinder".
      • Each shape may have an arbitrary number of goals.

          Example
          new Rules.Scale(Vec3(1.0, 2.0, 1.0)).Fulfills("Tall");

      • Split

        • Splits the shape.

          Constructors
          • new Rules.Split(Axis a, params Part[] shapes);
            a
            axis
            shapes
            shapes
          • new Rules.Split(Func<Attributes, Axis> a, Func<Attributes, params Part[]> shapes);
            a
            axis
            shapes
            shapes
          Description

          The split rule divides the shape into several shapes by slicing it along the given axis. It is the most essential rule for the grammar.

          Example 1
          new Axiom("A");
          //First we make the shape larger in the x-direction:
          new Rules.Scale(Vec3(6, 1, 1))
              .Fulfills("A")
              .Goal("S");

          //Basic Split:
          new Rules.Split(Axis.X,
          //The first part will be a shape with a new goal X.
          //Its size along the split direction will be 0.1/(0.1+0.2+0.3) = 1/6 of the size of the original shape.
                  Relative(0.1).Goal("X"),
                  Relative(0.2).Goal("Y"), //The second part will be a shape with a new goal Y, scaled to 2/6 of the original shape.
                  Relative(0.3).Goal("Z")) //The third part will be a shape with a new goal Z, scaled to 3/6 of the original shape.
              .Fulfills("S");
          new Rules.Asset(Primitive.Sphere).Fulfills("X");
          new Rules.Asset(Primitive.Cylinder).Fulfills("Y");
          new Rules.Asset(Primitive.Box).Fulfills("Z");

          Example 2 - Absolute and relative sizing
          //Let us create three differently sized initial shapes and another split rule.
          //Notice the differences when the same rule is appliet to various input shapes.

          new Axiom("B1");
          new Axiom("B2");
          new Axiom("B3");

          new Rules.Concat(
                  new Rules.Translate(Vec3(0, -2, 3)),
                  new Rules.Scale(Vec3(2, 1, 1))) //short initial shape (sized 2)
              .Fulfills("B1")
              .Goal("B");

          new Rules.Concat(
                  new Rules.Translate(Vec3(0, 0, 3)),
                  new Rules.Scale(Vec3(4, 1, 1))) //medium initial shape (sized 4)
              .Fulfills("B2")
              .Goal("B");

          new Rules.Concat(
                  new Rules.Translate(Vec3(0, 2, 3)),
                  new Rules.Scale(Vec3(6, 1, 1))) //long initial shape (sized 6)
              .Fulfills("B3")
              .Goal("B");

          new Rules.Split(Axis.X,
                  Absolute(1.0).Asset(Primitive.Sphere), //ABSOLUTE sizing, i.e. the size of X will be excatly 1.0 unit in the x-direction
                  //Relative(0.5).Asset(Primitive.Cylinder),
                  Relative(1.0)) //since this is the only RELATIVE sized part, it adapts to the remaining size of the original shape
              .Fulfills("B");

          //TODO: Uncomment Relative(0.5).Asset... and play with the size argument to see how to absolute and relative parts interact
          //TODO: Try changing the modes to LocalMid to see the centering effect of the Mid-point pivoting

            counter
            counter

            index
          Description

          The repeat rule is very often used along with the split rule.
          Split divides the shape into a fixed number of shapes.
          In opposite, repeat divides into a suitable number of shapes. All of them with the same semantics and size.

          Examples
          new Axiom("A").Restrict(Source.This);
          //First we make the shape larger in the x-direction
          new Rules.Scale(Vec3(5, 1, 1))
              .Fulfills("A")
              .Goal("S");

          //The repeat rule divides the shape into several shapes by slicing it along the given axis.
          new Rules.Repeat(Axis.X,
                  Absolute(1.0).Goal("X"))
              .Fulfills("S");
          //All resulting shapes are sized 1.0.
          //If the size of the shape is not dividable by the size of the resulting shape, it gets adapted a little bit to fit better.
          //Any other semantics attributes (goals, tags, asset, material) get also copied to all resulting shapes.

          new Rules.Asset(Primitive.Sphere).Fulfills("X");

          //Let us experiment a bit with repeats for inputs of various sizes.
          for(int i = 5; i <= 40; ++i)
          {
              new Axiom("A" + i.ToString());
              new Rules.Concat(
                      new Rules.Scale(Vec3(i * 0.1, 1, 1)),
                      new Rules.Translate(Vec3(0, 0, i)))
                  .Fulfills("A" + i.ToString())
                  .Goal("S")
                  .Paint(Material.Yellow());
          }

          //Now we will use the repeat rule to create a pyramid of balls.
          int size = 7;
          for(int i = 1; i <= size; ++i) //for cycle for the floors
          {
              new Axiom("Floor", i.ToString());
              new Rules.Concat(
                      new Rules.Paint(Material.Blue()), //paint it blue
                      new Rules.Translate(Vec3(10, Math.Sqrt(3) * 0.4 * (i-1), -10)), //translate the floor (smaller y will press the balls more together)
                      new Rules.Scale(Vec3(size - i + 1, 1, size - i + 1), Mode.LocalMid), //scale according to the floor number
                      new Rules.Repeat(Axis.X, Absolute(1.0).Goal("divideZ"))) //divide in the X direction
                  .Fulfills("Floor")
                  .IfGoal(i.ToString());
              new Rules.Repeat(Axis.Z, Absolute(1.0).Asset(Primitive.Sphere))
                  .IfIs("Floor")
                  .Fulfills("divideZ", i.ToString()); //divide in the Z direction
          }

Modes

  • Coordinate system - enum - possible options: Mode.Global, Mode.Local, Mode.GlobalMid, Mode.LocalMid

    The rotation is by default performed in Mode.Local that is around the local pivot, i.e. the left-lower-back point of the scope.

    Example 1

    To illustrate the principle of modes we create two objects.
    The cylindrical shape will help to show the non-uniform scaling.

    new Axiom("S1", "local").Restrict(Source.This);
    new Axiom("S1", "global").Restrict(Source.This);

    Please note that the order of the transformations is important for the result.
    First we rotate the shape along the X axis (so that its top side looks towards the camera).
    Local scaling by Vec3(2, 5, 2) makes the cylinder longer along its local y-axis
    Thus, the circular base is scaled uniformly by a factor of 2.

    new Rules.Concat(
            new Rules.Rotate(Axis.X, Deg(90), Mode.LocalMid),
            new Rules.Scale(Vec3(2, 5, 2)), //The local scaling mode is applied by default
            new Rules.Translate(Vec3(-3, 0, 0)))
        .Paint(Material.Red(2)) //pink
        .Asset(Primitive.Cylinder)
        .Fulfills("S1", "local");

    Scaling along the global pivot and along the global axes makes the cylinder longer in the direction of the GLOBAL y axis.

    new Rules.Concat(
            new Rules.Rotate(Axis.X, Deg(90), Mode.LocalMid),
            new Rules.Scale(Vec3(2, 5, 2), Mode.Global),
            new Rules.Translate(Vec3(-3, 0, 0)))
        .Paint(Material.Red(5)) //red
        .Asset(Primitive.Cylinder)
        .Fulfills("S1", "global");

    Example 2

    Now we create three spirals to demonstrate the effects of the various modes.
    The first one applies a translation to the shape and then rotates it around its local pivot point.

    for (int i = 0; i < 36; ++i) {
        new Axiom("LocalRotations" + i.ToString());
        new Rules.Concat(
                new Rules.Translate(Vec3(5, 0, 1.5*i)),
                new Rules.Rotate(Axis.Z, Deg(10 * i), Mode.Local))
            .Fulfills("LocalRotations" + i.ToString())
            .Goal("LocalCylinder");
    }
    new Rules.Translate(Vec3(-10, 0, 0)).Fulfills("LocalCylinder").Asset(Primitive.Cylinder).Paint(Material.Green(9));

    The second spiral applies the same translation, but then it rotates the shapes around the global pivot.
